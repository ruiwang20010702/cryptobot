## 三、P15 Volatile 策略优化（已完成）

### 背景

VPS 部署后运行 26 轮分析全部 no_trade。根因：FG=5（极度恐惧）强制升级 regime 为 volatile，但实际市场是有序空头趋势（H=0.62, ADX=58）。volatile_fear 仅允许 funding_arb + grid，但 `enabled: false` 阻断一切。结果：明显做空机会全部错过，780 次 LLM 调用浪费。

### 4 项修复

| 改动 | 文件 | 效果 |
|------|------|------|
| FG 升级趋势保护 | `collect.py` | H>0.55 + ADX>25 时不升级 volatile，保持 trending 允许做空 |
| funding_arb 绕过 enabled | `funding_arb.py` | volatile_mode 调用时跳过全局 enabled 开关 |
| volatile_fear 做空 fallback | `trade.py` | 无套利信号 + 趋势空头 → 保守做空 (1x, 80%置信度) |
| LLM 早期终止 | `graph.py` | 全部 observe/funding_arb/grid 时跳过 analyze/research 省 30 次 LLM |

## 四、P17 做多策略诊断与不对称约束（已完成）

### 背景

360d 回测暴露做多策略长周期下完全失效：做多 -$10,850 vs 做空 +$11,376。三层根因：Prompt 多空对称无做多特异性、置信度校准不区分方向（做多校准倒挂）、加密市场结构性做空优势（资金费率隐税/爆仓不对称/波动率偏斜）。

### 10 项改动

| 编号 | 改动 | 文件 | 效果 |
|------|------|------|------|
| P17.1 | 做多诊断交叉表 | `analytics.py` | 月×方向/置信度×方向/杠杆×方向三维诊断 |
| B1 | 做多杠杆上限 2x | `settings.yaml` + `risk.py` | 做多杠杆独立于全局 5x，钳位至 2x |
| B2 | direction_bias 硬性拦截 | `risk.py` | 策略路由要求仅做空时，硬性拒绝做多 |
| B3 | Kelly 三维查表 | `position_sizer.py` | (regime, confidence, is_long) 做多缩放降 ~30% |
| B4 | 做多 30d 胜率门控 | `risk.py` | <30% 暂停做多，<40% 杠杆降级 |
| A1 | 方向分拆置信度校准 | `analytics.py` + `confidence_tuner.py` | long/short 独立校准 + 做多专属阈值 |
| A2 | 做多绩效预警注入 | `analytics.py` | 做多胜率 <40% 时注入 Prompt 预警 |
| A3 | TRADER 做多特异性标准 | `prompts.py` | 多源共振/更紧止损/更高置信度要求 |
| A4 | trend_direction 路由 | `strategy_router.py` + `trade.py` + `graph.py` | trending+下降趋势 → direction_bias:short |
| 测试 | 29 个单元测试 | `test_p17_long_constraints.py` | 覆盖全部 B1-B4/A1/A4 改动 |

## 五、P16 Telegram 双向交互（已完成）

### P16.0 命令查询 + 生命周期通知

11 个 Telegram 命令 + 信号/平仓/日报/错误生命周期通知：
- `/status` `/signals` `/positions` `/balance` `/liq` `/pnl` `/edge` `/weights` `/risk` `/alerts` `/help`
- 信号/平仓/告警/日报自动推送

### P16.1 Freqtrade 离线自动 Fallback 虚拟盘

`/positions`、`/balance`、`/liq` 三个命令依赖 Freqtrade API。自动检测 Freqtrade 是否在线，在线用实盘数据，离线 fallback 到虚拟盘（funding_arb + grid）。标注 `[虚拟盘]`，用户跑实盘后无需配置改动。

### P16.2 离线体验优化

解决 Freqtrade 离线时"系统在工作但用户看不到"的问题：
- `/status` — 新增"运行模式"行（实盘/虚拟盘），有信号时标注"待执行"
- `/positions` — 无持仓但有信号时显示信号摘要（方向/币种/杠杆/入场区间）
- `/liq` — 无持仓但有信号时显示"N 个信号待 Freqtrade 执行"
- `/alerts` — FT 离线+有信号时追加 INFO 级别离线告警

### 核心约束

- 虚拟盘优先：所有新策略先跑 4 周虚拟盘，验证后才允许接入实盘
- 最大回撤 91.7% 是实盘不可接受的，需通过仓位控制和策略分散降至 <30%
- 高波动市策略（P14/P15）需特别注意滑点和流动性风险
